What's happenning here?
We are using react to get one element from the dom and creating a new element into it.
React.createElement takes as arguments the tag string, a dictionary of element attributes and the children of the element. So, a react element definition kind of reflects the actual dom.

The first element is inserting a simple anchor string

The second element is inserting two spans as children

The third one is generating a new plan ES6 class that can be reused and inserting it later. We are sending the `stranger` as properties.

This file is loading react and babel directly on the head!
